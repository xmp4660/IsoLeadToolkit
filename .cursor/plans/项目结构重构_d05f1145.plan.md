# 项目结构重构计划

## 当前问题分析

### 1. 目录结构混乱

- **根目录文件过多**：20+ 个 Python 文件全部放在根目录
- **对话框分散**：`file_dialog.py`、`sheet_dialog.py`、`two_d_dialog.py` 等 7 个对话框文件无统一管理
- **panel 相关模块分散**：`panel_utils.py`、`panel_handlers.py`、`panel_dialogs.py`、`panel_export.py` 在根目录，而 `panel_tabs/` 已是独立目录

### 2. 命名不一致

- `two_d_dialog.py` vs `three_d_dialog.py`（下划线 vs 无下划线 `2d` / `3d`）
- 函数命名混乱：`get_*` vs `select_*`
- 类命名：公开类 vs 私有类（`_Select2DColumnsDialog`）

### 3. 职责不清 & 代码耦合

- `main.py` 职责过多（初始化、配置、事件绑定、会话管理）
- `visualization.py` 过大（2000+ 行），包含算法、绘图、分析工具
- `geochemistry.py` 与 `V1V2_reference.py` 存在重复算法

### 4. 潜在循环依赖

- `visualization.py` ↔ `events.py` 可能存在循环依赖风险

## 关键代码变更

### 1. Import 路径更新示例

```python
# 旧代码
from config import CONFIG
from state import app_state
from data import load_data

# 新代码
from core.config import CONFIG
from core.state import app_state
from data.loader import load_data
```

### 2. 对话框基类示例 (`ui/dialogs/base.py`)

```python
import tkinter as tk
from tkinter import ttk
from core.localization import translate

class BaseDialog:
    """所有对话框的基类"""
    def __init__(self, parent, title_key):
        self.result = None
        self.dialog = tk.Toplevel(parent)
        self.dialog.title(translate(title_key))
        self.dialog.transient(parent)
        self.dialog.grab_set()
    
    def show(self):
        self.dialog.wait_window()
        return self.result
```

### 3. 精简后的 main.py

```python
from core.app import Application

if __name__ == "__main__":
    app = Application()
    app.run()
```
